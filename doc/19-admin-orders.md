## 筛选标签

| 标签 | 说明 |
|------|------|
| 全部 | 显示所有订单（默认选中，橙色高亮） |
| 待审核 | 用户已上报支付凭证，等待管理员确认 |
| 待支付 | 用户下单但尚未支付 |
| 已支付 | 支付成功的订单 |
| 已过期 | 超过支付时限的订单 |

- 标签样式：圆角胶囊按钮
- 选中态：橙色/金色背景 + 白色文字
- 未选中态：深色背景 + 浅色边框

## 订单表格列

| 列名 | 内容 | 说明 |
|------|------|------|
| 订单号 | `ord_20260611...` 格式 | 唯一订单标识 |
| 商品 | 商品名称 + product_id + 用户ID | 包含商品中文名、英文ID、下单用户 |
| 金额 | 实际支付金额 + 原价 | 显示折后价和原价 |
| 状态 | 待支付/待审核/已支付 | 彩色标签 |
| 通道 | 个人码 / 收款方 | 支付渠道信息 |
| 下单时间 | MM/DD HH:MM | 下单时间 + 过期时间 |
| 操作 | 确认到账 / 拒绝 | 管理员操作按钮 |

## 订单状态详解

### 待支付（橙色标签）

- 用户已创建订单但未完成支付
- 显示原价
- 操作：仅有"确认到账"按钮
- 示例价格：¥3.80、¥2.90、¥3.72 等

### 待审核（红色标签 ⚠️）

- 用户已上报支付凭证（截图/转账记录）
- 显示"上报 MM/DD HH:MM"时间
- 操作：**确认到账** + **拒绝** 两个按钮
- 需要管理员人工审核

### 已支付（绿色标签 ✅）

- 支付已确认完成
- 显示"上报 MM/DD HH:MM"时间
- 备注显示"管理员确认到账"
- 无操作按钮（已完成）

## 支付通道

- **通道**：个人码（个人收款码）
- **收款方**：菩提苑
- 支持用户上报支付凭证 → 管理员审核确认

## 价格策略观察

- 所有商品都有折扣（实际支付 < 原价）
- 折扣幅度约 2%~5%
- 例如：原价 ¥3.90 → 实际 ¥3.71~¥3.82
- 宝宝起名：原价 ¥66.00 → 实际 ¥65.97（几乎无折扣）

## 订单号格式

```
ord_YYYYMMDDHHMMSS_随机8位字符
例：ord_20260611153852_1c5b70
```

- 时间戳精确到秒
- 后缀为 8 位十六进制随机字符串

## 操作流程

### 确认到账流程

1. 用户下单 → 生成订单（待支付）
2. 用户扫码支付 → 截图/转账
3. 用户上报支付凭证 → 状态变为"待审核"
4. 管理员审核 → 点击"确认到账"
5. 状态变为"已支付"，备注"管理员确认到账"

### 拒绝流程

1. 管理员审核发现异常
2. 点击"拒绝"按钮
3. 订单状态变更（可能变为已过期或需要用户重新上报）

## UI 细节

- 表格使用深色背景 + 浅色分割线
- 状态标签颜色：待支付（橙色）、待审核（红色+⚠️）、已支付（绿色+✅）
- 操作按钮："确认到账"（绿色边框）、"拒绝"（红色边框）
- 订单号使用等宽字体
- 商品名称加粗显示
- 用户ID使用灰色小字
- 价格显示：实际价格大号 + 原价删除线小号
- 时间显示：下单时间 + 过期时间
