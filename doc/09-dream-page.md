# 周公解梦功能详解

## 路由

`/dream/` → 周公解梦页面

## 页面结构

```
┌─────────────────────────────────────────┐
│  # 周公解梦                              │
│  百梦皆有意 · 古今相参证                  │
├─────────────────────────────────────────┤
│  请描述您梦中所见                         │
│  ┌─────────────────────────────────┐    │
│  │                                 │    │ ← 梦境输入框
│  │                                 │    │
│  └─────────────────────────────────┘    │
│  [解梦]                                  │ ← CTA 按钮
├─────────────────────────────────────────┤
│  ## 按类查梦                             │ ← 分类浏览
│  [分类1] [分类2] [分类3] ...              │
└─────────────────────────────────────────┘
```

## 交互流程

1. 用户描述梦境内容（文本输入）
2. 点击 "解梦" 按钮
3. AI 分析梦境含义
4. 返回解梦结果（可能包含：吉凶判断、传统解释、现代心理学解读）

## 按类查梦

- 预设梦境分类（如：动物、自然、人物、物品、场景等）
- 点击分类可浏览常见梦境及解释

## API 接口（推测）

```
POST /api/v1/dream/interpret
Body: { description: "用户描述的梦境..." }
Response: {
  code: 0,
  data: {
    summary: "简要判断",
    traditional: "传统解梦含义",
    modern: "现代心理学角度",
    lucky: "吉" | "凶" | "平",
    advice: "建议"
  }
}

GET /api/v1/dream/categories
Response: {
  code: 0,
  data: {
    categories: [
      { id: "...", name: "...", icon: "...", dreams: [...] }
    ]
  }
}
```

## UI 细节

- 梦境输入：大文本框（textarea），placeholder 引导
- "解梦" 按钮：金色 CTA
- 分类浏览：可能使用卡片网格或标签列表
- 解梦结果：可能使用 Card 变体展示
