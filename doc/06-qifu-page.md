# 祈福功能详解

## 路由

`/qifu/` → 为家人祈福页面

## 页面结构

```
┌─────────────────────────────────────────┐
│  # 为家人祈福                            │
│  点一盏灯，挂家人之名，愿心愿成就，福寿安康。 │
│  已点亮 0 盏  今日新增 0 盏              │ ← 统计数据
├─────────────────────────────────────────┤
│  ## 为谁祈福                            │
│  家人姓名与您的关系：                      │
│  [父亲] [母亲] [爱人] [孩子]              │ ← 关系选择
│  [孙辈] [朋友] [自己]                     │
│                                         │
│  选一盏灯                                │ ← 灯选择器
│  供奉时长                                │ ← 时长选择
│  心愿（可选，最多 80 字）                  │ ← 心愿输入
│  您的称呼（可选，会显示在灯墙）             │ ← 称呼输入
│                                         │
│  需供奉 ¥6.6                            │ ← 价格显示
│  [点亮此灯]                              │ ← CTA 按钮
├─────────────────────────────────────────┤
│  ## 功德灯墙                            │
│  姓名已脱敏处理 · 心诚则灵                │
│  加载中...                               │ ← 灯墙展示
└─────────────────────────────────────────┘
```

## 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 关系 | 选项组 | 是 | 父亲/母亲/爱人/孩子/孙辈/朋友/自己 |
| 灯类型 | 选择器 | 是 | 多种灯类型可选 |
| 供奉时长 | 选择器 | 是 | 供奉持续时间 |
| 心愿 | 文本输入 | 否 | 最多 80 字 |
| 称呼 | 文本输入 | 否 | 显示在灯墙 |

## 商业逻辑

- 单盏灯价格：**¥6.6**
- 功德灯墙展示已点亮的灯（姓名脱敏）
- 统计数据：已点亮总数、今日新增数

## API 接口（推测）

```
POST /api/v1/qifu/create     ← 创建祈福订单
POST /api/v1/qifu/wall       ← 获取灯墙数据
POST /api/v1/qifu/stats      ← 获取统计数据
```

## 交互流程

1. 选择关系（按钮组高亮）
2. 选择灯类型
3. 选择供奉时长
4. 填写心愿（可选）
5. 填写称呼（可选）
6. 点击 "点亮此灯" → 跳转支付
7. 支付完成后灯出现在灯墙

## UI 细节

- 统计数字使用 `font-display` 金色
- 关系选择器：按钮组，选中态高亮
- 灯类型选择器：可能为图片/图标选择
- 心愿输入：`textarea`，带字数限制
- "点亮此灯" 按钮：金色 CTA，可能使用 `variant="ritual"`
